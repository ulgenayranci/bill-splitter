## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`px-6`, `gap-6`) |
| xl | 32px | Layout gaps (`gap-8`) |
| 2xl | 48px | Major section breaks (bottom padding `pb-24` reserve for fixed footer) |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: minimum 44x44px (Apple HIG). Person chips use `min-h-12 min-w-12` (48px). `ClaimableItemCard` must have `min-h-[44px]` added explicitly.
- Safe area insets: all sticky footers use `style={{ paddingBottom: 'env(safe-area-inset-bottom, 16px)' }}` in addition to padding tokens. This requires `viewportFit: 'cover'` in `app/layout.tsx` (Wave 0 fix).

Source: verified from `AssignItemsStep.tsx`, `ResultsStep.tsx`, `GuestClaimingView.tsx` — existing pattern consistent across all sticky footers.

## Interaction Contracts

### Copy Button Label Swap (D-05)

State machine: idle → copied (2000ms) → idle.

```
"Copy summary"  ──[tap]──►  "Copied!"  ──[2000ms]──►  "Copy summary"
     idle                    active                        idle
```

- Use `useState<boolean>(false)` for `copied` state.
- `setTimeout(() => setCopied(false), 2000)` after clipboard write.
- Clipboard icon: `<Copy />` from lucide-react alongside label (Claude's discretion — recommended to include, matching `HostWaitingScreen.tsx` copy button which already uses `<Copy />` + `<Check />` icon swap).
- Silent catch on clipboard write failure (non-fatal, per established pattern in `HostWaitingScreen.tsx`).

Source: D-05; RESEARCH.md Pattern 2; `HostWaitingScreen.tsx` lines 23, 42-49.

### Optimistic Claim Revert (D-08)

- Optimistic update applied immediately on tap.
- On network error: revert optimistic state, set `itemErrors[itemId] = true`.
- Inline error label appears on affected row only.
- Error clears on next successful tap of the same item.
- No global error banner, no toast.

Source: D-08; RESEARCH.md Pattern 3.

### Safe Area Insets (D-10)

All sticky/fixed footers must include:
```
style={{ paddingBottom: 'env(safe-area-inset-bottom, 16px)' }}
```

Prerequisite: `app/layout.tsx` must export:
```typescript
export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  viewportFit: 'cover',
}
```

Without `viewportFit: 'cover'`, `env(safe-area-inset-bottom)` returns `0` on iOS — the CSS is already present in all sticky footers but has no effect until the viewport export is added.

Source: D-10; RESEARCH.md Pattern 4 and Pitfall 1.

### Touch Target Minimum (D-12)

All interactive elements: minimum 44x44px per Apple HIG.

| Element | Current State | Required Fix |
|---------|--------------|-------------|
| Person chips in `AssignItemsStep` | `min-h-12 min-w-12` (48px) — satisfies | None |
| `ClaimableItemCard` tap area | Implicit via padding — not explicit | Add `min-h-[44px]` to Card className |
| "I'm done" button | `h-12 w-full` (48px) | None |
| Trash icon buttons | `h-12 w-12` (48px) | None |
| Confirm checkmark buttons | `h-12 w-12` (48px) | None |

Source: D-12; RESEARCH.md Pattern 5.

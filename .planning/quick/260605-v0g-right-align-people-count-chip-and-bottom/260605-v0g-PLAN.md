<objective>
Close the two cosmetic placement gaps from Phase 7 round-2 UAT:

1. The people-count chip should be right-constrained to the far end of the
   "Who's involved in the split?" heading row (currently sits immediately after
   the label, before the divider line).
2. The "Continue to Assign" button should be anchored to the bottom of the Setup
   screen rather than flowing inline directly below the people section — while
   preserving the existing disabled-gating logic (`canContinue` = billScanned &&
   people.length >= 2).

No behavior/schema changes. Pure layout. Existing tests must stay green.
</objective>

<context>
Current SetupStep heading row (components/wizard/SetupStep.tsx ~line 261):
```
<div className="flex items-center gap-2">
  <span ...>Who&apos;s involved in the split?</span>
  <span data-testid="people-count-chip" className="rounded-full bg-zinc-100 ...">{people.length}</span>
  <span className="h-px flex-1 bg-zinc-200" />   {/* divider eats remaining space */}
</div>
```
The chip sits BEFORE the flex-1 divider, so it hugs the label. Moving it AFTER
the divider pins it to the right edge.

Current layout container chain:
- WizardShell `<main className="flex-1 px-6 py-8 pb-24">` — grows to fill the
  viewport height inside a `min-h-screen flex flex-col` parent, but is a plain
  block so its child (SetupStep root) only takes content height and sits at top.
- SetupStep root `<div className="flex flex-col gap-5">` — content-height only.
- Continue wrapper `<div className="mt-2" style={{paddingBottom: 'calc(env(safe-area-inset-bottom) + 16px)'}}>`
  — flows right after the people section.

To bottom-anchor reliably, use the canonical sticky-footer pattern: make `<main>`
a flex column, make the SetupStep root a flex-1 child that fills it, and push the
Continue wrapper down with `mt-auto`. Only one wizard step renders inside `<main>`
at a time and all step roots are full-width block/flex columns, so adding
`flex flex-col` to `<main>` is layout-neutral for the other steps (Assign,
Results) — they keep content-height and top alignment because they don't set
`flex-1`.
</context>

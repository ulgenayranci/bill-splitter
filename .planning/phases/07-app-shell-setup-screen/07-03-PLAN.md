## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model → server | gpt-4o-mini returns a currencyCode string that is untrusted; the route validates it against `^[A-Za-z]{3}$` and falls back to 'USD' before returning it to the client. |
| server → client | The OCR response { items, currencyCode } crosses to the browser; only a normalized 3-letter code is emitted, never raw model text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-03-01 | Tampering | currencyCode from model output | mitigate | Route regex-validates `^[A-Za-z]{3}$` and upper-cases; any non-conforming value (symbols, empty, long strings) is replaced with the 'USD' default before reaching the client/store. |
| T-7-03-02 | Information disclosure | OCR error path | mitigate | On any thrown error the route returns generic `{ error: 'OCR failed' }` and logs server-side only — provider internals (incl. any currency-inference detail) never leak (carried from Phase 2 T-2-04). |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/ocr | SetupStep sends a user-supplied receipt image (compressed base64) to the OCR route. The route validates the data-URI shape and never echoes provider internals (handled in Plan 07-03). |
| browser → camera | `<input capture="environment">` accesses the device camera; the captured File is a blob URL then a base64 data-URL held only in client state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-02-01 | Denial of service | SetupStep image upload | mitigate | Image is compressed client-side to maxSizeMB 0.5 / maxWidthOrHeight 1920 before upload; OCR route additionally caps image length at 10MB (Plan 07-03). |
| T-7-02-02 | Information disclosure | billImageUrl persistence | accept | Photo persists as a base64 data-URL in localStorage only on the user's own device (per useBillStore partialize); no transmission beyond the OCR call. |
| T-7-02-03 | Tampering | Continue gate bypass | accept | Gate is a UX guard (`disabled` + hint); downstream Assign/Results recompute from store state, so a forced step change cannot fabricate items or people. |
</threat_model>

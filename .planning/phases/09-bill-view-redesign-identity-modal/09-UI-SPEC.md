## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip ring-offset, inline badge padding |
| sm | 8px | Gap between people-strip chips, gap inside avatar chip |
| md | 16px | Card horizontal padding (`px-4`), modal body padding, header left/right padding |
| lg | 24px | Section vertical padding (`py-6`), identity modal inner content gap |
| xl | 32px | Gap between major sections within the Bill View |
| 2xl | 48px | Page-level vertical breathing room (top of item list to header) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: **44px minimum** height on all tappable elements (`min-h-[44px]` on cards, `h-11 w-11` on stepper buttons). Source: existing `ClaimableItemCard.tsx` pattern.
- Bill View header row: **48px** tall to accommodate the title + date block and people strip stacked comfortably. The AppHeader above it stays at 48px (`h-12`) per existing `AppHeader.tsx`.
- People strip pill height: **32px** (`h-8`) for the expanded own-identity pill; compact avatar circles: **32px** (`h-8 w-8`). Derived from design screenshot proportions.

|------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` — white | Page background (`bg-background`), card backgrounds, modal backdrop surface, Bill View header background, identity modal background |

Accent reserved for: own-identity pill, claimed-card tint + border, claimed-item check + qty count, hamburger menu bars, primary action buttons. NOT used on unclaimed items, other people's chips, or neutral copy.

## Interaction Contract

### Flows

**Initial identity selection (IDENT-01/02)**
1. User navigates to `/split/[sessionId]`
2. SWR fetches session; loading guard shows `<div role="status">Loading…</div>`
3. After session loads: read `localStorage.getItem('split:{sessionId}:personId')`
4. If no stored identity → open `IdentityModal` immediately (no close-X)
5. If stored identity AND valid slot still taken → restore `selectedPersonId`, skip modal
6. If stored identity but slot no longer in `personSlots` (edge case) → re-show modal

**Change identity (IDENT-03, D-03)**
1. User taps anywhere on people strip
2. `IdentityModal` opens with `showCloseButton={true}` (user already has identity, can cancel)
3. User picks different name → `selectedPersonId` updates, `localStorage` updates
4. Claims of old name stay unchanged (D-04)

**Claim single item (tap-to-join, D-13)**
1. User taps unclaimed single-qty item card → calls `handleShareChange(itemId, true)`, fires `POST /claim { action: 'share', joining: true }`
2. Optimistic update shows own-claim highlight immediately
3. User taps own-claimed single-qty item → `handleShareChange(itemId, false)`, fires `POST /claim { action: 'share', joining: false }`

**Claim multi-qty item (D-14 — unchanged stepper)**
1. Tap card → claims 1 unit (existing behavior)
2. +/− buttons adjust quantity (existing stepper, unchanged)

**"I'm done" with all items claimed**
1. Button tap → directly advances to tip/results (phase machine: `handleDone` → `'tip'`)

**"I'm done" with unclaimed items (D-09)**
1. Button tap → unclaimed warning Dialog opens
2. "Go back" → Dialog closes, user stays in claiming view
3. "Continue anyway" → phase machine advances to `'tip'` then `'results'`; "some items unclaimed" note on results

**Unclaimed banner tap (D-10)**
1. Banner tap → `scrollIntoView({ behavior: 'smooth' })` on first item where `totalClaimedQty < item.quantity`
2. Does NOT open any dialog

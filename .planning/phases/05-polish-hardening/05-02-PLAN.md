## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → Clipboard API | Plain-text totals written via `navigator.clipboard.writeText`. User-controlled person names appear in the output. |
| Client → /api/session | Existing share endpoint; this plan changes only the error-handling UX, no new network surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Information Disclosure | Copy summary clipboard write | accept | Clipboard write is initiated by an explicit user gesture (button click). Browser handles permission. Contents (person names + amounts) are user-supplied; no PII beyond what the user already entered locally. No risk of leaking session tokens or API keys. |
| T-05-02-02 | Tampering | Copy summary text construction | accept | Text built by string template with `formatCents` (numeric, deterministic) and person names rendered as plain text into a `string[]`. No injection surface — the output is plain text destined for clipboard, not interpolated into DOM. |
| T-05-02-03 | Repudiation | ShareLinkButton inline error replaces toast | accept | Error visibility is improved (inline-permanent vs ephemeral toast). Reduces UX confusion when share fails. |
| T-05-02-04 | Denial of Service | setTimeout for label revert | accept | Single 2000ms timeout per click; no accumulation (click triggers a single timer; rapid double-click only refreshes the timer state). Bounded by user gesture frequency. |

</threat_model>

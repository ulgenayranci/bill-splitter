<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /api/session/[id]/edit | Untrusted item mutations (name/price/qty/remove). No auth by design (CLAIM-03). |
| client → POST /api/session | Untrusted people/items/currencyCode at session creation. |
| client → POST /api/session/[id]/claim | Untrusted claim writes; concurrent-write race on multi-qty items. |
| route → Redis (Lua eval) | Server-authoritative atomic mutation boundary for claims. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-03 | Tampering | POST /edit input | mitigate | Port edit-request's V5 validation verbatim: name non-empty, priceCents/quantity/newPriceCents/newQuantity integer > 0, itemId must exist in session.items. Reject otherwise with 400. |
| T-08-04 | Information Disclosure | /edit + /session 500 paths | mitigate | Return generic `{ error: 'Edit failed' }` / `{ error: 'Session creation failed' }`; never spread the caught error into the response (reuse Phase 2/6 generic-error pattern). |
| T-08-05 | Elevation of Privilege | Removal of host gating on edit/claim | accept | Intended per CLAIM-01/03 — any participant may mutate items and claim. No NEW high-severity exposure: the /edit route validates input the same way the old edit-request route did, just without the approval queue. |
| T-08-06 | Tampering | edit_quantity below current claims | mitigate | Reject (400) when newQuantity < totalClaimed (Pitfall 4) so the claims structure can't be driven invalid; client prompts user to unclaim first. |
| T-08-07 | Denial of Service / race | concurrent last-unit claims | mitigate | Preserve the CR-03 atomic bounds-check loop inside QTY_CLAIM_SCRIPT (Lua eval) — do NOT regress to a pre-Lua GET-then-check, which reintroduces the race. |
| T-08-08 | Tampering | currencyCode field | mitigate | Validate `/^[A-Z]{3}$/`, default 'USD'; an arbitrary client string cannot poison display logic (Phase 10 reads a constrained code). |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed (RESEARCH: internal refactor, zero new deps). No slopcheck checkpoint required. |
</threat_model>

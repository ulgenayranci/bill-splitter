## Screen Inventory

This phase introduces or updates the following surfaces:

### 1. Results Screen (replaces PersonResultsScreen)

**Purpose:** Locked per-person breakdown; all participants visible; current user expanded by default.

**Layout:**
- Full-screen `max-w-[480px] mx-auto min-h-screen bg-background`
- Sticky header row: "Results" title (20px semibold) + back-to-tip chevron (ghost icon button)
- Body: scrollable list of PersonResultCard components (accordion)
- Current user's card: expanded by default, amber-accented total, itemized line items visible
- Other person's card: collapsed by default (name + total only), tap to expand
- Grand Total row: at bottom of card list, `text-[16px] font-semibold`, `border-t border-border`, full bill total
- Bottom action bar (fixed, same pattern as CollaborativeClaimingView): Copy, Edit, New Split buttons

**States:**
- Default: current user expanded, others collapsed
- Expanded other person: accordion opens to show their itemized line items
- Copy in-flight: Copy button shows `<Check size={16} />` icon for 2s then reverts

### 2. Tip Modal (replaces TipScreen full-page screen → Dialog)

**Purpose:** Add a tip percentage from the Results screen without leaving it; totals update immediately.

**IMPORTANT CHANGE from existing TipScreen:** TIP-02 requires "via a modal launched from the Results screen". The current TipScreen is a full-page route. Phase 10 converts it to a shadcn Dialog overlay.

**Layout:**
- `<Dialog>` with `<DialogContent>` — max width constrained to 480px, centered
- `<DialogHeader>`: title "Add a tip?" (20px semibold)
- Tip amount display: 28px semibold `text-amber-600`, label "Your tip" below in 14px `text-zinc-500`
- Preset buttons: three `Button variant="outline" h-11` side-by-side (10%, 15%, 20%)
- Custom % input: `<Input inputMode="numeric" />` with "Custom %" label (14px)
- Separator
- "Your total" row: 16px semibold, shows subtotal + tip
- Footer: "Confirm tip" Button `h-12 w-full bg-amber-600 hover:bg-amber-700`
- Error state: `role="alert"` paragraph `text-[14px] text-destructive` inline above footer

**States:**
- Closed: tip total in Results screen shows $0.00 or existing tip value
- Open: modal overlays results
- Submitting: Confirm tip shows `<Loader2 size={16} className="animate-spin" />`, button disabled
- Error: inline alert under tip total row
- Success: modal closes, Results screen totals update immediately (SWR mutate)

### 3. Currency Display threading (all amount surfaces)

**Purpose:** All `formatCents()` call sites receive the session's `currencyCode` so JPY, EUR, etc. render correctly.

**Affected surfaces this phase:**
- PersonResultsScreen / Results Screen total and line items
- TipScreen / Tip Modal amounts
- Any grand total or per-person total elsewhere in the `split/[sessionId]` view

**formatCents signature change:**
`formatCents(cents: number, currencyCode?: string): string`
- When `currencyCode` is omitted or `'USD'`, behavior is identical to current (backward-compatible, preserves all wizard call sites)
- Uses `Intl.NumberFormat(undefined, { style: 'currency', currency: currencyCode })` for formatting
- Zero-decimal currencies (JPY, KRW, etc.) display no decimal places — handled automatically by Intl.NumberFormat
- Source: STATE.md "formatCents gains optional currencyCode param (backward-compatible)"

**Currency fallback (CURR-03):**
- When `currencyCode` is absent from session payload or is an unrecognized code, default to `'USD'`
- No blocking UI — the session proceeds with the fallback currency
- If user wants to override: a small "Currency: USD (change)" inline button near the grand total opens a minimal input or select (discussed below in Copywriting)

## Interaction Contracts

### Results Screen — Accordion behavior

- Only one card can be expanded at a time (single-expand accordion)
- Tap on a collapsed card: expand it, collapse the previously expanded card
- Tap on the already-expanded card: it stays expanded (do not collapse — the current user should not accidentally collapse their own data)
- Exception: if the user manually taps their own already-expanded card, it may toggle (implementation discretion), but the default open state is expanded on mount

### Tip Modal — Preset + Custom interaction

- Tapping a preset (10/15/20%) sets both the displayed tip amount AND populates the custom % field
- Typing in custom % field overrides any preset selection (no visual "selected" state on presets)
- Empty custom field OR "0" → tip = $0.00 (no tip)
- Custom % cap: 100% (established in TipScreen via MAX_TIP_PERCENT = 100); values > 100 are silently clamped to 100
- Confirm is always enabled (even at $0.00 tip) — user can skip tip by tapping Confirm with 0%

### Copy Summary

- Button: "Copy summary" (see Copywriting section)
- Clipboard content format (plain text):
  ```
  {Name} owes {formatted_amount}
  {Name} owes {formatted_amount}
  ...
  Total: {formatted_total}
  ```
- Amounts use the session's currency formatting (same as on-screen)
- Success feedback: button icon swaps to `<Check size={16} />` for 2 seconds, then reverts
- If clipboard API unavailable: execCommand fallback (same pattern as BillViewHeader.tsx handleShare)

### Edit Bill

- "Edit" button navigates back to the Bill View (claiming phase) for the current user
- This requires sending `done: false` to `/api/session/[sessionId]/done` first (established pattern in CollaborativeClaimingView `handleBackToClaiming`)

### New Split

- Clears local session context (localStorage key for this sessionId), redirects to `/` (home/Setup screen)
- Does NOT delete the Redis session — other participants may still use it
- Confirmation dialog: "Start a new split? This will clear your local progress." with Cancel / New Split buttons

## Accessibility Contracts

- Result cards: each card is a `<button>` or wraps a clickable `<div role="button" tabIndex={0}>` with `aria-expanded` state reflecting open/closed
- Expanded person section: `aria-label="{Name}'s breakdown"` on the trigger
- Tip modal: `<DialogTitle>` provides accessible name; `<DialogDescription>` provides context
- Loader state: Confirm tip button has `aria-label="Confirming tip…"` when submitting
- Error alerts: `role="alert"` on inline error paragraphs (established pattern in TipScreen + CollaborativeClaimingView)
- Copy button: `aria-label="Copy summary to clipboard"`; when copied, `aria-label="Summary copied"`
- Keyboard: all interactive elements reachable via Tab; modals trap focus (Radix Dialog handles this)

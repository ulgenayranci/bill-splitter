<objective>
Add an `update_currency` op to the existing `/api/session/[sessionId]/edit` route so a participant can change the session-level `currencyCode` and have it apply to the whole bill across all devices (D-07, CURR-03). This is the shared write path the Results-screen currency override (Plan 03) calls. The op follows the route's existing GET→mutate→SET pattern; no Lua atomicity is needed because `currencyCode` is a session-level singleton, not a concurrent-write-prone per-person key (last-write-wins is correct here).

Purpose: Persist a currency override so it is shared, not per-device — the requirement D-07 explicitly mandates.
Output: Extended `/edit` route + `update_currency` test block in `__tests__/editRoute.test.ts`.
